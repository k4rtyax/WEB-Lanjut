# Codeigniter4 - Session

1. Selanjutnya website akan dibatasi aksesnya, hanya user yang sudah berhasil login yang bisa mengakses menu-menu yang ada di dalam sistem. Jadi, diawal user harus menginputkan pasangan data username dan password yang dianggap valid oleh sistem pada sebuah form login. 
    
    Kemudian, data user yang sudah berhasil login tersebut harus disimpan dalam sebuah session, karena pada dasarnya protokol HTTP tidak menyimpan keadaan (stateless). Sehingga walaupun user sudah berhasil login pada halaman login, lalu user klik menu yang lain dan berpindah ke halaman lain (misalnya ke halaman produk), maka pada halaman baru itu nanti user akan kembali dianggap belum login. Oleh karena itu, perlu ada nilai-nilai yang disimpan pada sebuah session sebagai penanda user yang mengakses sudah berhasil login atau belum. 
    
    Session adalah mekanisme penyimpanan data sementara di sisi server yang digunakan untuk mempertahankan informasi user selama mereka berinteraksi dengan sebuah website. Setiap user akan memiliki identitas unik (session ID) yang biasanya disimpan di browser (melalui cookie), sementara data utamanya disimpan di server. Session yang ada pada Codeigniter pada dasarnya adalah session bawaan dari PHP yang dibungkus dalam sebuah Library, sehingga lebih mudah untuk digunakan. 
    
    Pertama buka file app/Views/components/header.php, cari code yang menampilkan menu Sign Out. Pada tag link nya <a>, ganti isi atribut href yang awalnya tanda pagar (#) menjadi **logout**. Route untuk logout akan dibuat pada langkah selanjutnya.

    Penjelasan fungsi yang digunakan :
    
    **form_open** adalah fungsi pembuka (awal) form, disini ditentukan action nya adalah login, sehingga nanti harus membuat route dengan nama login dengan method post, yang berguna untuk mengirimkan data dari form login menuju controller yang bertugas untuk memproses data tersebut.
    
    **form_input** adalah fungsi untuk membuat inputan form dengan tipe teks, yang isinya adalah variable $username yang sudah dibuat pada langkah no 5.
    
    **form_password** adalah fungsi untuk membuat inputan form dengan tipe password, yang isinya adalah variable $password, yang sudah dibuat pada langkah no 5.
    
    **form_submit** adalah fungsi untuk membuat inputan form dengan tipe submit, biasanya berupa tombol yang digunakan untuk mengirim data yang diinputkan pada form ke rute yang ditentukan pada bagian form_open.
    
    **form_close** adalah fungsi penutup (akhir) form.

9. Pada file app/Controllers/AuthController.php tambahkan fungsi __construct yang isinya adalah pemanggilan helper **form.** Pada helper form terdapat fungsi-fungsi yang dapat membantu dalam pembuatan dan penggunaan form (contohnya yang digunakan pada langkah no 7).
    
    ```php
    function __construct()
    {
        helper('form');
    }
    ```
    
    ![image.png](attachment:3b4dad50-aa83-42d0-9955-3a2c01900d70:image.png)
    
    Construct adalah fungsi yang pertama kali di jalankan pada saat sebuah class di akses, dalam hal ini adalah class AuthController.
    
    Tambahkan fungsi baru bernama login, untuk menampilkan halaman login dan juga memproses data username dan password yang dikirim melalui form login. 
    Untuk data user, sementara masih pakai data sample statis pada variable $dataUser (belum ambil dari database). **Username : april, Password :123, Role : admin.**
    
    Password dienkripsi menggunakan algoritma md5, jika ingin ganti password yang lain bisa menggunakan generator online misalnya https://www.md5hashgenerator.com/
    
    ```php
    public function login()
    {
        if ($this->request->getPost()) {
            $username = $this->request->getVar('username');
            $password = $this->request->getVar('password');
    
            $dataUser = ['username' => 'april', 'password' => '202cb962ac59075b964b07152d234b70', 'role' => 'admin']; // passw 123
    
            if ($username == $dataUser['username']) {
                if (md5($password) == $dataUser['password']) {
                    session()->set([
                        'username' => $dataUser['username'],
                        'role' => $dataUser['role'],
                        'isLoggedIn' => TRUE
                    ]);
    
                    return redirect()->to(base_url('/'));
                } else {
                    session()->setFlashdata('failed', 'Username & Password Salah');
                    return redirect()->back();
                }
            } else {
                session()->setFlashdata('failed', 'Username Tidak Ditemukan');
                return redirect()->back();
            }
        } else {
            return view('v_login');
        }
    }
    ```
    
    ![image.png](attachment:3b4b54eb-62f2-48ce-be31-0cdef5a87206:image.png)
    
    Alurnya, ketika ada data yang dikirim (post) dari form login, lakukan pengecekan apakah data username yang dikirim dari form cocok dengan data username yang tersimpan pada variable $dataUser. 
    
    Jika username tidak cocok, arahkan kembali ke form login dan tampilkan pesan failed 'Username Tidak Ditemukan'. Namun jika username cocok, lanjutkan dengan pengecekan password. 
    
    Jika password tidak cocok, arahkan kembali ke form login dan tampilkan pesan failed 'Username & Password Salah'. Jika password cocok, set variable session dengan isi username, password, dan role dari user yang berhasil login, lalu arahkan tampilan ke halaman utama.
    
    Pesan kesalahan login yang disimpan dalam session flashdata dengan index ‘failed’ akan ditampilkan pada bagian form login yang sudah dibuat pada langkah no 6 tadi.
    
    Kemudian, tambahkan fungsi baru bernama logout, code ini digunakan untuk mengakhiri sesi (destroy) lalu tampilan website kembali diarahkan ke form login.
    
    ```php
    public function logout()
    {
        session()->destroy();
        return redirect()->to('login');
    }
    ```

    Sehingga pada AuthController saat ini terdapat 3 fungsi yaitu : **__construct**, **login**, dan **logout**.
