Filter ini fungsinya seperti sebagai pelindung untuk controller. Terdapat sebuah class yan gmemiliki dua method yaitu before dan after yang digunakan untuk menyaring proses controller. Menyaring proses controller berarti nantinya proses filter akan dilakukan terlebih dahulu sebelum (before) melakukan proses controller maupun sesudah (after) melakukan proses controller.

Sederhananya,

**route > filter (before) > controller > filter (after)**

Logic yang akad dibuat adalah, sebelum user mengakses rute tertentu yang menggunakan filter Auth ini akan dicek dulu apakah user sudah berhasil login atau belum, jika belum maka akan diarahkan ke halaman login dulu sampai berhasil login (menginputkan pasangan data username dan password yang sesuai).

Maka pada filter auth yang dibuat, di bagian fungsi before ada pengecekan apakah pada session tidak terdapat data isLoggedIn. Karena pada saat user berhasil melakukan login, pasti pada session akan tersimpan data isLoggedIn yang bernilai true (lihat tutorial session langkah no 9). Jika pada session tidak terdapat data isLoggedIn maka akan diarahkan ke rute login.

Sedangkan pada fungsi after tidak melakukan apa-apa karena memang tidak perlu melakukan apa-apa setelah user berhasil mengakses rute.
